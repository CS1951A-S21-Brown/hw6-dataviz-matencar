Written questions:

1. My dashboard answers the presented questions by providing one visualization/graph for each question. To answer the first question, I took the “top video games” to be the video games with the most global sales. Thus, I presented a bar chart showing the top 10 most sold video games of all time, where the y-axis gives the name of the video game and the x-axis shows how many units sold there were for all-time. I also implemented a tooltip box when you hover your mouse over the bar, to give additional info on sale numbers for different regions.
To answer the second question, I interpreted the “most popular” genre as the genre with the most video game sales. Therefore, I separated sales per region, and used a pie-chart to represent the most popular genres for each region. The labels on the side show the top 3 genres for that region and the total sales for that genre in that region. If the user clicks on a button, they can see results for different regions.
To answer the third question, I interpreted the “top publisher” for each genre to be the publisher with the most video games published for that genre. Therefore, I showed a graph with the top 10 publishers with most video game titles published for that genre. The results are sorted with the top publisher at the top. The user can click on the buttons to see results for different genres.


2. One benefit of using d3 is that it provides an expansive selection of charts and graphs that you can implement. This helped me improve my visualization, as i wasn’t limited to using just one chart/graph type, but instead had a plethora of choices, which helped me choose the best chart for the situation. Another benefit of using d3 is that it has extensive documentation that is easy to follow (d3-graph-gallery), this made making the visualization much easier, as I had examples I could reference. One final benefit of using d3 is that it runs on javascript, which computationally isn’t too expensive and an easy language to read. Furthermore, javascript is supported by different web browsers which make d3 compatible with different web browsers


3.One reason why d3 may not be the best tool for creating a visualization is that the learning curve for learning how to use d3 can be quite steep. This is because some knowledge on javascript, HTML, web-dev is assumed, so if you don’t know much about these topics d3 might be challenging. Another reason why d3 may not be the best visualization tool is that it is a strictly web based tool, which means that d3 wouldn’t be a good option if you wanted to create visualization on a mobile app. Finally, the last reason why d3 may not be the best visualization tool is that other data visualization libraries may be easier to understand than d3. For example, dc.js is easier to use for some people than d3, as it is a library built on top of d3 and does not require as many calculations/preliminary set up when displaying charts. Thus dc.js may be more suitable if you need a basic visualization up quickly.


4. Users with colored, regular vision would find this dashboard accessible, as the dashboard conveys information solely through visual cues (there is no sound at all). This means that users who are blind will be unable to understand the dashboard without some additional assistance. Thus, I need to consider more about how assistive technology can be used with my dashboard. For example, using a suggestion from the medium article “Why Accessibility Is at the Heart of Data Visualization,” my charts would benefit from the addition of “text descriptors”, as it would be useful for a screen reader, consequently allowing blind people a better opportunity to understand what the chart is showing. In addition, the same article mentions how “two or more sensory techniques” can be utilized to make a data visualization more impactful. This is why I think some sort of sound aspect to my visualization would help reiterate its message, as well as make it more accessible to the visibly impaired. In addition, I think my visualization works well to convey information to English speakers, as all the text is in English. However, this would mean that non-English speakers would have trouble understanding the visualization. Thus, I would incorporate a feature that allows the user to change the language of the dashboard to a few other common languages (e.g. Hindi, Mandarin, and Spanish).
